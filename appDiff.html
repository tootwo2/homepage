App比较

概念

  1. web app，实际上就是网页，只是对移动端进行了优化。例如滔小微。
    • 优点
      • 跨平台
      • 较低的开发成本
      • 可以实时更新发布
    • 缺点
      • 对联网的要求比较大
      • 用户体验没那么炫酷(无法充分利用硬件性能)
      • 对一些硬件的调用受限制，比如摄像头和存储
  2. native app(原生开发)，基于不同的操系统开发。分别使用android和IOS系统的系统接口开发。订采就是原生开发。
    • 优点
      • 可访问手机所有功能（GPS、摄像头）
      • 速度更快、性能高、整体用户体验不错
      • 可离线使用
    • 缺点
      • 开发成本高，周期长
      • 更新系统比较麻烦

  3. 混合开发，结合native app和web app，高翎的做法就是这样，微信开发也是，腾讯做了native app，第三方(像我们公司)的企业在进行web开发

    • 特点
      • web app和native app的优势互补

  4. 使用跨平台技术，同一套代码编写两个系统的程序（坑比较多）。类似小程序(比较特殊)，weex这样的。

混合开发的优势

  1. 首次加载以后，页面打开非常快。
    • 加载的所有资源可以通过离线方式缓存，再次打开是不需要重新加载
    • 微信企业号开发，本身是就是网页开发，腾讯目前还不开放离线的功能，每次都是需要重新加载页面素材，速度明显慢
  2. 可以离线使用
    • 在没有网络的情况下，可以访问本地资源，比如后台数据暂时出问题，可以优先访问历史数据
    • 微信企业号，如果没有网络就无法使用
  3. 用户体验更好（大部分app都是能通过使用明显区分的）
    • 混合开发，部分组件可以使用原生组件，用户体验更好，更加流畅
    • 网页受限于容器，能调用的资源受限制，部分效果不是很好。
